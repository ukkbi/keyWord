# RAM

## 주 기억 장치(Random Access Memory)

## 역할

ssd에 저장된 데이터를 저장해 놨다가 CPU로 주고 받는 역할을 함

## 특징

전원이 꺼질 시엔 Ram에 저장된 데이터는 사라진다

Ram의 내부 공간은 수많은 셀로 쪼겨져 있고 이 셀들은 고유의 주소값을 갖는다

## 작동 방식

1. 프로그램의 실행
2. CPU 작업에 필요한 데이터를 SSD에서 Ram으로 복사
3. Ram에 적힌 주소를 바탕으로 CPU에서 빠른 속도로 데이터를 읽고 쓰면 연산을 처리
4. 프로그램 종료 시 Ram 내부에 있던 데이터는 지워져 공간이 확보됨

## 클럭 속도 및 대역폭

Ram이 초당 처리할 수 있는 데이터를 나타내는 지표.

MHz단위로 표기하고, DDR5에선 MT/s(MegaTransfers per Second)라는 단위를 사용

## 램 타이밍

Ram이  CPU의  명령을 받고 데이터를 찾아 보내는 데 걸리는 지연시간

CL(CAS Latency)00-00-00-00 같은 형식으로 표기됨

## 가상 메모리

실행하려는 프로그램의 크기가 실제 물리 Ram 용량보다 클 때 SSD의 일부 공간을 Ram처럼 쓰는 것

## 가비지 컬렉션 및 메모리 누수

메모리 누수 : 프로그램이 종료 되었을 때에도 Ram에 저장된 데이터가 반환되지 않고 쌓여있는 버그

가비지 컬렉션 : 개발자가 따로 비우지 않아도 가상 머신이 더 이상 쓰지 않는 Ram 데이터를 청소해주는 

자동 메모리 관리 시스템

# VRAM

GPU 전용의 고속 메모리

## RAM과 분리되어 있는 이유

매초 수백번 씩 연산해야 하는 데 이 계산 결과 값을 CPU와 RAM의 통로로 주고 받을 시 병목현상이 발생

따라서 GPU  전용으로 대역폭이 극단적으로 넓은 전용 메모리를 붙여 초고속으로 데이터를 처리하게 함

## VRAM에 저장되는 데이터의 종류

1. 텍스쳐 
2. 메쉬 및 지오메트리 : 3D 모델의 뼈대와 형태를 구성하는 Vertex(정점), index(인덱스) 데이터
3. 셰이더 : 화면의 정점과 픽셀을 어떻게 계산할지 정의한 GPU용 프로그램 코드
4. 프레임 버퍼 : GPU가 연산을 끝내고 모니터로 출력하기 직전의 완성된 화면 이미지를 임시로 저장하는 곳

## 텍스쳐 압축

GPU가 디코딩 없이 직접 읽을 수 있는 특수한 포맷으로 이미지를 압축하는 기술

## 가상 텍스쳐링

VRAM의 용량 한계를 극복하기 위해 보이는 화면만 실시간으로 VRAM 올리고 안 보이는 곳을 내리는 기술
