# 파일 포맷

기계어를 어떤 방식으로 해석할지에 대한 운영체제와 프로그램 사이의 약속

## 확장자와 포맷의 차이

확장자는 표시해주는 이름표에 불과하고 파일 내부의 데이터 구조가 파일 형식을 결정한다

확장자를 바꿨다고 해서 파일 포맷이 바뀌진 않는다

## 프로그램의 파일 포맷 구별법

파일의 맨 앞부분을 열어보는데 이 부분에 고유의 식별코드로 구별을 하게된다. 이 코드를 매직 넘버라고 한다

### 예)

- PNG 89 50 4E 47
- PDF 25 50 44 46
- ZIP 50 4B

## 파일 포맷 예시

- JSON : 웹,앱, 게임 개발에서 데이터를 주고받을 때 쓰는 표준 포맷
- XML : 태그<tag> 형식을 사용하여 데이터를 구조화 하는 포맷
- YAML : Unity 설정 파일이나 도커, 서버환경 설정에 많이 쓰이는 포맷(들여쓰기 기반이라 가독성이 좋음)
- PNG : 파일의 손상이 없는 그래픽 포맷, UI나 투명도가 필요한 곳에 사용한다
- JPG : 파일 압축률이 높은 그래픽 포맷, 배경 이러스트 같은 곳에 사용한다
- FBX : 3D 모델링 데이터를 게임 엔진으로 가져올 때 많이 사용되는 3D 파일 포맷
