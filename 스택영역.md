# 스택영역

가장 효율적이고, 빠르며, 엄격한 규칙에 의해 움직이는 공간

## 함수의 임시 메모리 저장소

함수가 실행되는 동안만 잠깐 살다가 함수가 끝나면 사라지는 데이터가 저장된다

### 저장대상

- 지역변수 : 함수 내부에서 선언된 변수들
- 매개 변수 : 함수를 호출할 때 넘겨받는 인자 값들
- 복귀주소 : 함수 실행이 끝난 후 원래 코드로 돌아가야 할 코드 영역의 메모리 주소

### 데이터 타입과 관계

C# 이나 C++에서 구조체, 정수, 실수, bool 같은 값 타입 변수는 스택 영역에

참조타입(class)변수를 선언하면 힙 영역에 저장되고 주소값만 스택에 저장됨

## 특징

1. 후입선출 : 가장 나중에 실행된 함수가 스택의 가장 위에 쌓이고 함수의 실행이 끝나면
맨 위에서부터 차례대로 제거된다
2. 힙 영역과 정 반대로 높은 주소에서 낮은 주소 방향으로 데이터를 채운다
3. 컴파일러가 코드를 분석할 때 몇 바이트의 메모리가 필요한지 미리 계산할 수 있다
4. 스택 오버플로우 : 스택 영역은 속도가 매우 빠르지만 전체 용량이 매우 작다
이 공간이 다 차게됐을 때 프로그램이 터지는 현상
