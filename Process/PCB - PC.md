# PCB - PC

다음번에 CPU를 다시 잡으면 실행해야 할 명령어의 메모리 주소를 기록해 두는 값

## 특징

- 실행 시점의 저장소 역할
    - CPU 내부에도 물리적 PC 레지스터가 존재함, CPU가 동작하는 동안에는 이 물리 레지스터가 
    매 클럭마다 다음 실행할 코드 주소를 가리킴
    - 하지만 문맥 교환이 일어나 프로세스가 쫓겨나면, CPU 내부의 PC 레지스터 값은 증발하거나
    다른 프로세스의 주소로 덮어씌워짐.
    - 따라서 OS는 쫓겨나는 프로세스의 마지막 PC 레지스터 값을 해당 프로세스의 PCB메모리 영역에 안전하게 저장함
- 복원 및 재개
    - 스케줄러에 의해 해당 프로세스가 다시 선택되면, OS는 PCB에 적어두었던 PC 값을 CPU의 실제 PC 제지스터로 복사해 옴
    - 프로세스는 처음부터 재시작할 필요 없이, 중단됐던 바로 그 코드의 다음 줄부터 자연스럽게 
    이어 실행 할 수 있음
