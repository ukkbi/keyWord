# 코드 영역

개발자가 작성한 코드를 컴파일러를 이용해 기게어로 변환한 결과물이 저장하는 곳

## 특징

1. 읽기 전용 : 프로그램이 실행되는 동안 내용을 수정할 수 없도록 함
(실행중에 바뀐 코드의 버그 발생, 해킹을 통한 악성 코드 주입 방지)
2. 정적 구조 : 컴파일이 완료되는 순간 명령어 개수와 크기가 정해짐, 
따라서 실행 중에 용량이 늘어나거나 줄어들지 않는다
3. CPU와의 직통 대역폭 : CPU 안에 레지스터가 코드 영역에 있는 기계어를 한 줄씩 가져와
ALU를 통해 실행 

[CPU(CentralProcessingUnit)](https://app.notion.com/p/CPU-CentralProcessingUnit-389a94a44953806793a1f3c4cc763ee7?pvs=21)

1. 코드의 재사용과 공유
똑같은 프로그램을 2개 동시에 실행 할 때 내용이 변하지 않는 코드 영역은 메모리에 하나만 올려두고 
두 프로세스가 물리 메모리를 공유하게 함
2. 빌드 결과물의 크기 : 빌드한 실행 파일의 용량 중 대부분이 코드 영역의 크기, 
무거운 외부 라이브러리를 많이 포함할수록 코드 영역이 차지하는 RAM 용량이 커짐
