# 힙 영역

프로그램 실행 중에 필요에 따라 자유롭게 메모리를 할당하고 해제 하는 동적할당 구역

## 특징

- 프로그램이 실행되는 도중에 데이터의 크기가 유동적으로 변할 때 사용된다
- 저장되는 대상
    - C#에서 new 키워드로 생성하는 모든 클래스의 인스턴스
    - 배열, 리스트, 문자열 등 크기가 유동적으로 변할 수 있는 참조 타입 데이터
- 메모리의 효율적 사용을 위해 메모리의 낮은 주소에서 높은 주소로 데이터를 쌓아나간다
- 힙 영역의 데이터는 이름표가 없음, 그래서 데이터를 만들게 되면 
해당 데이터의 메모리 주소를 스택 영역에 있는 변수에 담아서 제어함

## 메모리 누수

동적으로 할당한 힙 메모리 사용 후 시스템에 반환하지 않아 메모리가 낭비되는 현상

C, C++의 경우엔 개발자가 직접 해제해줘야 함

## 가비지 컬렉션(GC)

GC가 힙 영역을 청소할 때에 데이터가 꼬이는 것을 막기 위해 프로그램의 모든 동작을 숝간적으로 멈춤

그래서 new 키워드를 남발하게 되면 렉이 발생하게 된다

## 메모리 단편화

힙 영역에 객체들을 무작위로 생성했다가 지우기를 반복하게 되면 메모리 사이에 빈 공간이 생기는 현상

빈 공간에 크기에 맞지 않는 용량이 큰 배열이 끊겨 저장되어 프로그램이 튕기는 현상이 생길 수 있다
