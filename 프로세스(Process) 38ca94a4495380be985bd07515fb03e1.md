# 프로세스(Process)

## 프로그램과 프로세스의 차이

프로그램 : SSD 같은 저장장치에 존재하는 정적인 코드와 데이터의 집합

프로세스 : 시스템 RAM에 적재하고 CPU를 할당 받을 수 있는 상태

## 프로세스의 메모리 구조

1. 코드 영역 : 소스 코드가 기계어로 변경되어 저장되는 구역
2. 데이터 영역 : 전역 변수와 정적변수가 저장되는 구역
3. 힙(Heap) 영역 개발자가 필요에 따라 **동적으로 메모리를 할당**하는 구역
(프로그램이 종료되도 사라지지 않음, 참조 형식의 원본 값을 저장하는 구역)
4. 스택(stack) 영역 : 지역 변수, 매개변수, 리턴 주소 등이 **임시**로 저장되는 구역

## PCB(Process Control Block)

## 프로세스 제어 블록

1. PID : 프로세스 고유 번호
2. 프로세스 상태 : 현재 프로세스의 상태 정보
3. PC(Program Counter) 프로세스가 다음번에 실행해야 할 코드의 메모리 주소를 가르키는 포인터
4. CPU 레지스터 정보  : CPU에서 연산하던 중간 값들을 백업해 두는 공간

## 프로세스 와 스레드의 차이

프로세스 : 운영체제로부터 독립된 메모리를 통째로 할단 받는 대단위 작업 단위

한 프로세스가 크래시로 죽어도 다른 프로세스에 영향을 주지 않는다

스레드 : 코드를 실행하는 역할이고 프로세스가 가진 메모리 중 스택 영역을 제외한 부분을 공유

스레드는 고유한 스택 영역을 따로 가짐 하나의 스레드가 죽으면 프로세스 전체가 죽을 수 있음
