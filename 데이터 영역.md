# 데이터 영역

프로그램이 실행되는 동안 변하지 않고 유지되어야 하는 값들을 모아두는 곳

## 저장되는 대상

- 전역 변수 : 어떤 함수나 클래스 외부에서 선언되어 프로그램 어디서나 접근할 수 있는 변수
- 정적  변수 : static 키워드가 붙어 객체를 여러개 만들어도 메모리에 하나만 생성되어 공유되는 변수

## 데이터 영역의 세부구역

1. Data 영역
    - 특징 : 소스 코드에서 초기값을 명시해준 전역,정적 변수가 저장된다
    - 원리 : 변수들의 초깃값 정보는 실행 파일에 그대로 포함되어 저장되어야 하기에
    프로그램 용량을 차지함
2. BSS 영역
    - 특징 : 초깃값을 지정하지 않았거나 0, null로 초기화한 전역, 정적 변수가 저장되는 곳
    - 원리 : 이 영역의 변수들은 컴파일러가 실행 파일에 실제 값을 저장하지 않고 
    크기 정보만 기록하여 실행 파일의 용량을 줄여주는 최적화 기술이다.

## 작동 방식

데이터 영역은 힙, 스택 영역과 지속적으로 상호작용한다

1. 프로그램 실행
2. 데이터 영역에 변수 저장
3. 힙 영역에 객체 생성 
4. 데이터 영역의 변수가 힙 영역의 주소를 가르키게 됨

## 싱글톤 패턴과 메모리 상주

게임 내 요소를 관리할 매니저를 만들 때 static을 쓰게 되는데 싱글톤 객체에 너무 많은 데이터를 저장할 시
RAM 용량을 과도하게 차지하므로 주의 필요
