# Thread

프로세스에서 실제로 작업을 실행하는 가장 작은 단위

하나의 프로세스는 하나 이상의 스레드를 가질 수 있음

프로세스가 할당받은 자원을 활용하여 명령어를 순차적으로 실행함

## 특징

- 자원 공유 구조 : 스레드는 프로세스 메모리 영역 중 Code, Data, Heap 영역을 같은 프로세스의
다른 스레드와 공유하고 각자 고유한 Stack 영역과 CPU  Register세트만 독립적으로 보유
- 가벼운 흐름
    - 프로세스를 새로 만드는 것에 비해 생성 및 종료, 스위칭 비용이 훨씬 적음
    - 스레드 간 컨텍스트 스위칭은 메모리 주소 체계를 바꿀 필요가 없어 
    CPU 캐시 메모리를 그대로 유지 할 수 있어 매우 빠름
