# 프로세스의 메모리 영역

## 구성

1. 코드 영역
    - 작성한 소스 코드가 컴파일 되어 CPU가 이해할 수 있는 기계어의 형태로 저장되는 곳
        - 생명주기 : 프로그램이 실행될 때 메모리에 로드되고 
        프로그램이 완전히 종료될 때까지 유지
        - 특징 : 실행 중에 프로그램 코드가 바뀌면 안되기 때문에 읽기 전용으로 보호됨
        - CPU가 이 영역에서 코드를 한 줄씩 가져가서 연산을 수행한다.
2. 데이터 영역
    - 프로그램 전체에서 사용되는 전역변수와 정적변수(static)가 저장되는 곳
    (너무 많은 대용량 데이터가 할당되어 있으면 비효율적 일 수 있음)
        - 생명주기 : 프로그램이 시작될 때 할당
        - **구조적 분류**
            1. Data 영역 : 초기 값이 있는 변수가 저장됨
            2. BSS(Block Started by Simbol) 영역 초기값이 없어 자동으로 null,0으로 초기화 되는 변수가 저장됨
3. 힙 영역
    - 개발자가 프로그래밍 언어의 문법을 통해 동적으로 공간을 할당하고 해제하는 곳
    (C/ C++의 경우엔 메모리를 할당 한 후에 해제하지 않으면 메모리 누수가 일어남)
        - 생명주기 : 코드로 생성하는 순간 할당됨 직접 해제하거나 가비지 컬렉터가 수거할 때 소멸
        (가비지 컬렉터가 너무 자주 작동하면 렉이 발생하므로 주의)
        - 특징
            - 럼타임 중에 크기가 결정됨
            - 메모리 주소가 낮을 주소에서 높은 주소 방향으로 쌓임
4. 스택 영역
    - 지역변수, 매개변수, 함수가 끝나고 돌아갈 리턴 주소등이 저장되는 공간
    (힙에 비해 할당된 크기가 아주 작음 무한 재귀 함수에서 벗어나지 못한다면 스택 공간이 가득차서 프로그램이 강제 종료되는 스택 오버플로우가 발생함)
        - 생명주기 : 함수가 시작될 때 할당, 끝날 때 메모리에서 자동 증발
        - 특징
            - 컴파일 시점에 대략적인 크기가 결정
            - 가장 나중에 들어온 데이터가 먼저 나가는 구조
            - 힙 영역과 구딪히지 않기 위해 높은 주소에서 낮은 주소 방향으로 쌓임
